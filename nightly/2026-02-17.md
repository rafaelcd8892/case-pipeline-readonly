# 2026-02-17

- Fixed a bug where some board items were linked with random IDs instead of the correct profile ID. This caused items to appear "orphaned" in queries.
- Updated documentation so motions are correctly described as standalone items, not paired with court cases.
- Added a CI pipeline that runs type checking and tests on every pull request.
- Made board item data queryable. Status, next date, attorney, and profile link are now real database columns instead of buried in a JSON blob. This means we can now answer questions like "show me all cases set for hearing" without parsing raw JSON.
- Added full-text search on client profiles. Searching by name, email, or phone is now instant.
- Built a query layer that assembles a complete 360-degree view of any client: their profile, active and closed contracts, cases across all boards, and appointment history.
- Added a `lookup` CLI command so you can search clients and see their full case picture from the terminal.
- Built a JSON API with 5 endpoints (search, client detail, contracts, board items, board item detail) served by Bun.
- Built a web dashboard using React and Tailwind CSS. You can search for a client by name and see their full 360 view — profile, contracts, active cases by board, and appointments — all in a clean, collapsible layout.
- The dashboard is config-driven. Adding or removing a board from the view is a one-line change in a config file, not a code change.
- 225 tests passing across 10 test files.
