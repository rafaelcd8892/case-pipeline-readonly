# 2026-03-07 — Bug Fixes, Appointments Upgrade, and Groups Foundation

## What was accomplished

**Fixed the Alerts dashboard card.** The card was showing a count of alerts but displaying "No active alerts — all clear" because it never loaded the actual alert items. Now the card shows the top 5 alerts across all severity levels (critical, warning, info).

**Fixed the "All" filter in Clients view.** Clicking "All" on the priority filter showed zero results because the API returned data in a different shape when no filters were active. The response is now consistent regardless of which filters are selected.

**Upgraded the Appointments page with new time ranges.** Previously, appointments could only be viewed for "Today" or "This Week." Added "Upcoming" (today and forward) and "All" (including past) options. Also fixed the seeder — it was generating all appointments in the past, so the daily view was always empty. Now 40% of seeded appointments are in the future.

**Created a graphical sitemap.** Added `docs/sitemap.md` showing every page, its controls, and how data flows from user actions through the frontend to the API and query layer. This will be updated as the app grows.

**Added Monday.com groups as first-class data (Schema v6).** In Monday.com, the group an item belongs to determines critical things — whether someone is a client or just a consultee, whether a case is active or closed, whether a form is filed or still in progress. The database now tracks this:
- Profiles have a `group_title` column (Active Clients, Non-clients, Closed clients, Clinic Profiles)
- Board items already had `group_title` — added a composite index for fast filtering
- Documented the semantic meaning of groups across all 18 boards

**Made group distributions realistic across every board.** Previously, every seeded court case was in "Court Case," every motion in "Motions to be sent," etc. Now each board distributes items across all its real groups with weighted randomness — court cases can be active, inactive, granted, or withdrawn; motions can be pending, awaiting decision, granted, or denied; and so on for all 12 board types. This makes the test data match what the real Monday.com data looks like.

## Why it matters

The first three fixes resolve usability issues that made key features appear broken. The groups foundation is the most important change — without knowing which group an item belongs to, the dashboard couldn't reliably answer basic questions like "Is this person actually a client?" The realistic group distributions mean the dashboard can now be tested with data that exercises every filtering scenario. When Monday.com sync is added, `item.group.title` from the API populates the same column — no code changes needed.
