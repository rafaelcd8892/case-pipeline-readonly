# 2026-02-18

## Client Updates & Notes Timeline

- Added a new `client_updates` table to store every update, reply, and automation email from Monday.com boards. Each entry tracks who wrote it, when, which board item it belongs to, and whether it's a top-level update or a threaded reply.
- Built a seeder factory that generates realistic update threads — including multi-reply conversations, @mentions, and automation-style emails — across all board types.
- Added a query function and API endpoint (`/api/clients/:id/updates`) that returns all updates for a client, sorted newest-first.
- The dashboard now shows a full Updates & Notes timeline at the bottom of the 360 view. Updates are grouped by date, show author initials with color-coded avatars, and indent threaded replies. Long histories collapse with a "show more" toggle.

## Real Profile Fixtures

- Added 4 handpicked real Monday.com profiles to the seeder: Ashik (criminal consult), Jabez (file copy with Fee K), Karen (detained habeas consult), and Fernando (notes-intensive 42-B COR case with 50+ court case updates).
- Built a reusable `fetch-profile.ts` script to pull any profile from Monday.com by item ID, including all linked items and their updates.
- Built a code generator (`gen-fernando-fixture.ts`) that converts a fetched JSON profile into a typed TypeScript fixture automatically — avoids hand-translating dozens of updates.
- These fixtures always seed alongside generated data so the dashboard shows real-world examples without needing a Monday.com connection.
